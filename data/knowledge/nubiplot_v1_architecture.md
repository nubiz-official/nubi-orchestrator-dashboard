# NUBiPLOT_V1 아키텍처 분석

**작성일**: 2026-03-26
**버전**: v1.0

---

## 1. 정체성

**NUBiPLOT_V1**는 누비즈의 전략/기획 분석 에이전트로, 복잡한 사업계획서·회의록·외부자료를 입력받아 누비즈 전체 역량과 교차 분석하여 **솔루션·할일·예산·일정**을 체계적으로 산출하는 AI 에이전트이다.

> 기존 **NuBI Orchestrator**(통합관리)와의 차이: Orchestrator는 프로젝트 인덱싱/분류/연결, Strategist는 **전략 분석/솔루션 도출/실행계획 수립**

## 3. 3-Layer 아키텍처 상세

### Layer 1: INPUT LAYER (자료 수집/파싱)

```
외부 자료 입력
├── PDF 파싱 (pdfplumber/PyMuPDF)
│   ├── HWP→PDF 변환 파일 처리
│   ├── UTF-8 인코딩 강제 적용
│   └── 전체 텍스트 추출 → .txt 저장
├── 인라인 문서 수신
│   └── 사용자가 직접 붙여넣은 자료 (회의록 등)
└── 프로젝트 내부 자료 스캔
    ├── _analysis/ 기존 분석 보고서
    ├── _intake/ 원본 자료
    └── _hub/knowledge/ 지식 베이스
```

**이번 세션에서 수집한 자료**:
| 소스 | 내용 | 규모 |
|------|------|------|
| 사업계획서 v4.3 PDF | 71페이지 전체 텍스트 추출 | ~2,700줄 |
| 회의록 PDF | 인라인 3페이지 | 회의 지적사항 3건 |
| _analysis/ 기존 분석 | document_summary, market_research, feasibility | 5개 파일 |
| _hub/knowledge/ | projects, technologies, people, links | 4개 파일 |
| memory/ | project_overview, user_profile | 2개 파일 |

**핵심 기술적 해결**:
- Windows 환경에서 `pdftoppm` 미설치 → `pdfplumber` Python 패키지 설치로 해결
- CP949 인코딩 충돌 → `python -X utf8` + `io.TextIOWrapper` 적용
- 대용량 PDF → 텍스트 파일로 중간 저장 후 분할 읽기

### Layer 2: KNOWLEDGE LAYER (교차 분석 엔진)

이것이 NUBiPLOT_V1의 **핵심 두뇌**이다.

```
교차 분석 엔진
│
├── ① 문제 식별 (Problem Extraction)
│   └── 회의록/지적사항에서 핵심 이슈 구조화
│       ├── 지적 1: AI 요소 부족
│       ├── 지적 2: 데이터 기업활용성
│       └── 지적 3: 실증 범위 모호
│
├── ② 원인 분석 (Root Cause Analysis)
│   └── 사업계획서 원문과 대조하여 근본 원인 도출
│       ├── 사업계획서의 어떤 부분이 부족한지
│       └── 어떤 내용이 빠져 있는지
│
├── ③ 역량 매칭 (Capability Matching)
│   └── 누비즈 전체 프로젝트/기술과 교차 매칭
│       ├── projects.md (P01~P18)
│       ├── technologies.md (AI/블록체인/바이오/인프라)
│       ├── people.md (핵심 인물/기관 관계)
│       └── 기존 서비스 포트폴리오 (25개 서비스)
│
├── ④ 솔루션 생성 (Solution Generation)
│   └── 문제 × 원인 × 역량 = 맞춤형 솔루션
│       ├── 누비즈가 직접 수행 가능한 것
│       ├── 다른 주체가 해야 하는 것
│       └── 협업이 필요한 것
│
└── ⑤ 실행계획 수립 (Execution Planning)
    └── 솔루션 → 항목/방법/일정/예산/담당 구체화
```

**교차 분석 매트릭스 (이번 세션 예시)**:

```
           │ AI파이프라인 │ 데이터카탈로그 │ 블록체인ALCOA+ │ 대시보드 │
───────────┼─────────────┼──────────────┼──────────────┼─────────┤
P02 노화센터│     ◎       │      ◎       │      ○       │    ◎    │
P07 블록체인│     ○       │      △       │      ◎       │    △    │
P12 DCP    │     ◎       │      ◎       │      ◎       │    ◎    │
기존서비스  │     ○       │      ◎       │      △       │    ◎    │
───────────┴─────────────┴──────────────┴──────────────┴─────────┘
◎ 직접 활용  ○ 연관 참조  △ 간접 참조
```

### Layer 3: OUTPUT LAYER (보고서 생성)

```
보고서 생성 엔진
│
├── 구조화된 마크다운 출력
│   ├── Part 1: 지적사항 분석 & 솔루션 (표 기반)
│   ├── Part 2: 전체 할일 목록 (카테고리별)
│   ├── Part 3: 주체별 할일 (6개 주체)
│   └── Part 4: 누비즈 상세 (항목/방법/일정/예산)
│
├── 파일 직접 쓰기 (Write Tool)
│   └── 긴 보고서는 화면 출력이 아닌 파일 저장
│       (Not Responding 문제 해결)
│
└── 요약 출력 (화면용)
    └── 핵심 테이블만 화면에 표시
```

## 4. 작동 프로세스 (이번 세션 기준)

```
Step 1: 사용자 프롬프트 수신
        "자료1 + 자료2 → 분석 → 솔루션 → 주체별 할일 → 누비즈 상세"
                                    │
Step 2: PDF 파싱 시도 (실패 → pdfplumber 설치 → 성공)
        71페이지 → 사업계획서_v4.3_fulltext.txt 저장
                                    │
Step 3: 누비즈 Knowledge Base 수집
        _analysis/(8파일) + _hub/knowledge/(4파일) + memory/(2파일)
                                    │
Step 4: 교차 분석 실행
        회의록 3대 지적 × 사업계획서 71p × 누비즈 18개 프로젝트
                                    │
Step 5: 보고서 생성 → 파일 직접 저장
        04_광주시지적사항_솔루션_주체별할일.md
                                    │
Step 6: 핵심 요약만 화면 출력
```

## 5. 강점과 한계

### 강점
| # | 강점 | 설명 |
|---|------|------|
| 1 | **교차 분석 능력** | 외부 자료 + 내부 18개 프로젝트 + 기술/인물 DB를 한번에 교차 분석 |
| 2 | **구조화된 산출물** | 지적→원인→솔루션→할일→예산까지 논리적 체인 |
| 3 | **누비즈 맥락 이해** | DCP, 블록체인, SmartBell 등 누비즈 전체 역량을 솔루션에 반영 |
| 4 | **다국어 문서 처리** | 한글 PDF, 영문 논문, 혼합 문서 모두 처리 |
| 5 | **적응적 문제해결** | PDF 파싱 실패 시 대체 라이브러리 설치, 인코딩 문제 자동 해결 |

### 한계 (개선 필요)
| # | 한계 | 원인 | 개선 방안 |
|---|------|------|---------|
| 1 | **대용량 출력 시 Not Responding** | 긴 텍스트를 화면에 직접 출력하면 타임아웃 | 파일 저장 후 요약만 출력 (해결됨) |
| 2 | **My Backups 미연결** | 클라우드 백업 스토리지 연결 안됨 | 로컬 동기화 또는 API 연결 필요 |
| 3 | **실시간 기업 데이터 부재** | 광주 AI 기업 1,810개사 실제 데이터 없음 | 외부 DB/API 연동 필요 |
| 4 | **예산 추정의 한계** | 시장가격 기반 추정이며 실제 견적은 아님 | 미소정보/디윅스 등 실제 견적 확인 필요 |

## 7. 호출 방법

사용자가 다음과 같은 요청을 하면 NUBiPLOT_V1가 작동:

- "이 사업계획서를 분석해줘"
- "광주시 지적사항에 대한 솔루션을 구해줘"
- "누비즈가 할 일을 정리해줘"
- "주체별 할일/예산/일정을 정의해줘"
- "DCP했던 것처럼 기획서+누비즈 전체를 학습하고 분석해줘"

**트리거 키워드**: 분석, 솔루션, 전략, 할일, 주체별, 예산, 사업계획서 보완
